# @r4pm/components

React components for process mining: Petri nets, object-centric Petri nets,
directly-follows graphs, dotted charts, alignments, etc.

## Install

```sh
pnpm add @r4pm/components
# peer dependencies:
pnpm add react react-dom
```

## Use

Wrap your app (or any single viewer) in a Radix `Theme`, import the stylesheet once, then
feed a viewer the typed binding result. Graph viewers also need a layout engine, since the core ships
none: pick one and supply it via `ViewerConfigProvider` (see
[Layout Engines](?path=/docs/getting-started-layout-engines--docs)).

```tsx
import "@r4pm/components/styles.css";
import { Theme, PetriNetViewer, ViewerConfigProvider } from "@r4pm/components";
import { elkLayout } from "@r4pm/components/elk-layout";

export function App({ net }) {
  return (
    <Theme>
      <ViewerConfigProvider value={{ layout: elkLayout }}>
        <PetriNetViewer data={net} />
      </ViewerConfigProvider>
    </Theme>
  );
}
```

Coloring, value formatting, interactivity, and the layout engine are shared across viewers and
configured once. See [Viewer Configuration](?path=/docs/getting-started-viewer-configuration--docs)
and [Layout Engines](?path=/docs/getting-started-layout-engines--docs). For the exact
shape of each viewer's `data` prop (the binding return types, with field descriptions), see
[Data Types](?path=/docs/getting-started-data-types--docs).
